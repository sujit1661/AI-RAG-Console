Here's a clean implementation:

def reverse_list(head):
  prev = None
  while head:
    nxt = head.next
    head.next = prev
    prev = head; head = nxt
  return prev
What's the time complexity?
O(n) time and O(1) space — we visit each node once with no extra data structures.

Questions

Frequently Asked Questions

Everything you need to know

What file formats are supported? expand_more
PDF, DOCX (Word), XLSX/XLS (Excel), PNG, JPG, and JPEG. Image files are processed using Groq's vision model to extract text. TXT, MD (Markdown), CSV, RST, and LOG files are also supported — read as plain text and chunked normally. Max file size is 50 MB.
Does it work without Supabase? expand_more
Yes — the app runs fully locally without any Supabase configuration. It uses ChromaDB for vector storage, BM25 pickle files on disk, and JSON files for chat/workspace data. Supabase is an optional cloud persistence layer that makes data survive server restarts and deployments.
How accurate are the answers? expand_more
Very accurate for factual queries within your documents. The system uses hybrid retrieval (vector + BM25 keyword search) fused with Reciprocal Rank Fusion, then optionally re-ranked by Cohere. The LLM is instructed to answer only from the provided context — it won't hallucinate from outside knowledge.
Can I upload multiple documents to one workspace? expand_more
Yes. Each workspace supports unlimited documents. When you ask a question, the AI searches across all documents in the workspace simultaneously. You can also create multiple workspaces to keep different projects separate.
Is my data private? expand_more
Your documents are stored in your own Supabase project — we never have access. Document text is sent to Groq for LLM inference (covered by Groq's privacy policy). All sessions use secure HttpOnly cookies and JWT authentication.
Can I deploy this myself? expand_more
Yes — the project includes a render.yaml for one-click Render deployment. You'll need a Groq API key (free), a Supabase project (free tier), and optionally a Cohere API key for reranking. Full setup instructions are in the README.
What is the Pipeline Explorer? expand_more
The Pipeline Explorer (/pipeline) is an educational step-by-step walkthrough of the full RAG pipeline. Upload any file, ask a question, and watch the system execute each stage — embedding bars, BM25 keyword scores, the RRF fusion table, Cohere reranking scores, and the final streamed LLM answer. You can ask multiple questions on the same file without re-uploading.
Can I see query analytics and system health? expand_more
Yes. The Analytics Dashboard (/dashboard) shows aggregate stat cards (workspaces, documents, chunks, queries, storage, feedback), a recent query log with color-coded latency, and a per-workspace overview. The System Monitor (/monitoring) shows real-time health cards for every service — Supabase, embedding model, BM25, ChromaDB, Groq, and Cohere — with latency bars, uptime counter, and a recent error log.
